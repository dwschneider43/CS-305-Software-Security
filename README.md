## Module Eight Journal – Portfolio Reflection

### Client Summary
Artemis Financial is a financial services company that needed help improving the security of its software. Their main concern was protecting sensitive financial data and making sure their application followed strong security practices. My role in this project was to identify vulnerabilities in their system and apply secure coding techniques to strengthen their overall security posture.

### What I Did Well
I feel that I did a solid job identifying the security issues in the application, especially when reviewing dependency vulnerabilities and configuration weaknesses. Using automated tools alongside manual review helped me catch problems that might have been easy to overlook. Writing secure code is important because it protects both the company and its customers, and it builds long‑term trust in the software.

### Challenges and Helpful Parts
One of the more challenging parts of the vulnerability assessment was sorting through the scanner results and determining which findings were real risks versus false positives. Even though it was challenging, it was also helpful because it pushed me to think critically about each vulnerability instead of relying only on automated output.

### Increasing Layers of Security
I increased security by updating outdated dependencies, implementing secure communication (HTTPS/TLS), and applying secure coding practices throughout the application. Going forward, I would continue using tools like dependency checkers, static analysis tools, and manual code reviews to assess vulnerabilities and choose the right mitigation strategies.

### Ensuring Functionality and Security
After refactoring the code, I tested the application to make sure everything was still working correctly. I also ran the vulnerability scans to confirm that no new issues were introduced. This combination of testing and scanning helped ensure the application stayed both functional and secure.

### Resources and Practices for the Future
Tools like OWASP guidelines, dependency‑check reports, and secure coding references were extremely useful. Practices such as version control, incremental testing, and documenting changes will continue to help me in future courses and real‑world projects.

### What I Can Show Employers
From this assignment, I can show employers a complete vulnerability assessment report or a secure coding practices report. These artifacts demonstrate my ability to analyze risks, apply secure coding techniques, and document my work professionally all skills that are valuable in software development and security roles.
